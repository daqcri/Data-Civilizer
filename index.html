Data Civilizer: A Robust Disguised Missing Values Detector

Why do I need FAHES:

Detectable DMVs are:

  • Out of range data values, e.g. disguise the missing values in an attribute that takes only positive values with a negative value.
  • Outliers, e.g. disguise the missing values with a very large value or a very small value such as replacing the missing age values with the value 1000.
  • Strings with repeated characters or characters that are next to each other on the used keyboard, e.g. replacing a phone number with 5555555555.
  • Values with non-conforming data types, e.g. disguising the missing strings with numerical values and vice versa.
  • Valid values that are randomly distributed within the range of the data and used frequently in the data set.

FAHES has three main modules:

  • Syntactic outlier detection module to capture those DMVs that are syntactic outliers or contain special patterns such as strings with repeated substrings (e.g. abcabcabc) and numbers with incrementally increasing/decreasing digits (e.g. 1234567)
  • Numerical outlier detection module that best suits FAHES. This module detects DMVs that are far from the rest of the values in the Euclidean space
  • DMVs as MCAR/MAR detector: for detecting DMVs as inliers, we devise an algorithm for detecting DMVs that follow missing-completely-at-random (MCAR) or missing-at-random (MAR) models
